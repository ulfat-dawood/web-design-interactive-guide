Desktop First Design

In desktop-first desing:

  1. Layout for large screens is designed first.
  2. Media query breakpoints for smaller screens are defined later.
  3. The goal is to scale down the layout and UI elements.
  • This design approach is followed when the website traffic is mostly coming from desktop computers.
  • Example:

Scale down the UI

Large-scale Layout (grid-based)

  • Reduce the number of grid columns.
  • Allow all the grid-areas or grid-items to fit into two or one column.

Small-scale Layout(flex-based)

  • Wrap flex-items to span multiple lines.
  • Change the direction of flex-container to row columns.
  • change the overflow of the container to scroll
  • change the order of flex items as needed

navigation Menus

  • It is recommended to always keep all UI elements visible and in display
  • But when navigation menus are too large to fit into mobile screens, hamburger menus can be applied as an alternative solution.

Illustrations

  • Icons are informative and used to convey some visual meaning.
  • Unlike icons, some illustrations on desktop view are there for decorative purposes to fillin white space.
  • To make the mobile view less crowded, these illustrations can be illiminated.
