CSS Rules

Normally, CSS rules consist of a selector followed by a set of decelerations that define styles for that selector. Yet, CSS rules can do more. bellow a list of other CSS rules:

  • @media
  • @keyframes
  • @import
  • @font-face
  • @charset

@media Rule

As discussed in the responsive design module, @media rule defines different styles for different media conditions.

@keyframes Rule

@keyframes rule defines the keyframes for CSS animation.
More about @keyframes in the next module.

@import Rule

  • @import rule allows to include CSS styles into the current file.
  • @import rule must be declared before all other rules
    except @charset .
  • It is important to keep the PRECEDENCE in mind, since the current file's styles will override the imported styles.

  • Example:
    • @import "./media-queries.css";
    • @import url("./media-queries.css"); ;

@font-face Rule

  • The @font-face rule allows to include fonts other than web save-fonts.
  • @font-face rule should be added to the stylesheet before any styles.

  • @font-face {
      font-family: myFont;
      src: url(sansation_light.woff);
    }

  • .myClass {
      font-family: myFont;
    }

@charset Rule

  • @charset rule defines the character encoding.
  • @charset rule must be the first rule defined in a the stylesheet.

  • Example:
    • @charset "UTF-8";
