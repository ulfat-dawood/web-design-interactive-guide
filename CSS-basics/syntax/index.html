Cascading Style Sheets (CSS)

REMEMBER:

  • The terms: Tag and Element are used interchangeably
  • In HTML nested tags have a parent / child relationship.
  • Parent elements have children, and children elements are also parents to the tags nested within them.

HTML vs CSS

  • HTML: provides only the hierarchical structure of elements in your document.
  • CSS: layered on top of that structure to modify visual appearance of those elements.

Adding CSS to HTML

Inline CSS

  • CSS declerations written as a value for the global attribute: style="set of CSS decleratios"

  • <p style="font-size:18px; color:blue;"> some text <p/>

precedence :

Highest priority, overrides other internal and external CSS styles.

Internal CSS

  • CSS rules written within the HTML paired tag:
    <style> CSS Rules <style/>

  • <style> tag is placed inside the <head> tag.

  • <head>
    <style>
    p{
    font-size:18px;
    color:blue;
    }

    #main-section{
    background: grey;
    }

    <style/>
    <head/>

precedence :

Second in priority, overrides external CSS styles.

External CSS

  • HTML self-closing tag: <link/>

  • CSS rules written in seprate file that has the extention .css

  • <link href="./style.css" type="text/css" rel="stylesheet" />

precedence :

If no inline or internal style found, external CSS style is applied.

Order   Counts

Order of Rules

  • The latter CSS rules override the previous ones.

p{color: blue;
 font-weight:bold;
 text-decoration:underline;
}

p{
  color:red
}


Only color property is overriden by the last rule.

Praragraphs color changed

*If order of rules is reversed, paragraphs would appear blue.

color not changed

Order of Declerations

  • The latter CSS declerations override the previous ones.

p{color: blue;     /*fallback value*/
  color: var(--blue);
}


The vairable var(--blue) will overeide the value blue

The previous decleration is used as a fallback value, in case some properties or values have browser compatibility issues or not supported.

Specificity

The more specific the CSS selector is, the higher is the precedence of the CSS property declarations.

Specificity hierarchy follow the following order

  1. anySelector{background:orange !important;}          more specific
  2. #id{background:red;}
  3. .class{background: blue;}
  4. element{background: pink;}
  5. *{background:white; }         least specific
