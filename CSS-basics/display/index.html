Display

Display Property

By default, display of elements is either block or inline.


  • display: block | inline | inline-block;

Block Elements

  • <div>
  • <hr>
  • <video>
  • <form>
  • <table> <tfoot>

  • Text containers: <h1> - <h6> <p> <pre>

  • Lists: <ol> <ul> <li> & <dl> <dt> <dd>

  • Semantic tags: <article> <header> <aside> <figcaption> <section> <figure> <main> <footer> <nav>

Inline Elements

  • <span>
  • <img>
  • <a>

  • Text formatting: <b> <i> <em> <sup> <sub>

  • form tags: <input> <label> <button> <textarea> <select>

  • Semantic tags: <strong> <q> <abbr> <cite> <acronym> <dfn>

Inline Elements

Quirk :

Inline elements are not allowed to have custom height and width.
Rather, height and width of inline lements will always wrap the content.
(unless display property is explicitly set to block or inline-block.)

Stacking

ROWS: elements stacked next to each other, while allowing elements that don't fit to wrap to the next line.

this is an anchor span another span element another span element

Quirk :

  1. While inline elements are allowed to have padding, they are not allowed to have margin-top or margin-botton.
  2. This causes overlapping of:
    • Inline element's border with the parent.
    • Inline element's border and padding with other siblings.

inline-block

Both block and inline elemets' display can be set to inline-block, by explicitly spesifying the following property

display:inline-block

Pros :

inline-block elements are allowed to have custom height and width.

Stacking

ROWS: elements are stacked next to each other, while allowing elements that don't fit to wrap to the next line.

Pros :

  1. inline-block elements are allowed to have both margin-top and margin-botton.
  2. There's no overlapping.
