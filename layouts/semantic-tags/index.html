Semantic Tags

What Are HTML5 Semantic Tags

  • Used to build the web layout.
  • Don't have any distinct visual appearance.
  • Help the browser understand what role is played by the content contained within the semantic tags.
  • Help improve Webpage SEO.
  • Using semantic tags is a good accessibility practice.

Block Level Semantics

Used in the place of <div> tags to break your page content into identified parts, each of which fulfills a specific role.
They are used to structure and build the page layout, and divide the page content into its basic parts

  • <header>: Introductory information of a page or section.

  • <nav>: Main navigation menu links

  • <main>: The body of a page. There should be only one <main> per page.

  • <article>: Self-contained content. e.g. a blog post.

  • <section>: Groups together nearby content of a similar theme.

  • <aside>: Content that is less important.

  • <footer>: Used at the base of a page or section.

  • <figure>: Illustrations, diagrams, photos...etc.

  • <figcaption>: caption for a <figure> element

  • <details>: Additional details.

  • <summary>: Heading for a <details> element

Inline Level Semantics

Used in the place of <span> tags

  • <mark>: Highlighted text

  • <time>: Date or time

  • <strong>: Content has strong importance.

  • <em>: Emphasize words

  • <q>: Shorter quotes

  • <blockquote>: Longer quotes

  • <abbr>: Abbreviations.

  • <cite>: Citation reference.

  • <dfn>: Terminology explaination.
